Official location to download Postgresql

Steps to setup a Docker container for Postgresql:

  1. Pull docker image:

    docker pull postgres

  2. Start the docker container:

    docker run -d --name postgres -e POSTGRES_PASSWORD=lsf -p 5433:5432 -v D:\DockerVolumes\postgresql:/var/lib/postgresql/data postgres

  3. Connect the Postgresql server to validate it:
    1. Open pgAdmin, in the left pane click: Register/Server
    2. In Connection tab, input port: 5433, password: lsf, as set in previous step.
    3. Create a table:

      CREATE TABLE employees (
         id SERIAL PRIMARY KEY,
         name VARCHAR(100) NOT NULL,
         role VARCHAR(50),
         salary DECIMAL(10, 2),
         hired_on DATE NOT NULL DEFAULT CURRENT_DATE
      );

    4. Delete the docker container.
    5. Check if the data files are persisted under D:\DockerVolumes\postgresql.
    6. Run the command in Step 2 to start the container again.
    7. Connect with pgAdmin again, check if the registered server is still valid, and the table employees still exists.

      select * FROM pg_catalog.pg_tables where tablename='employees';

Steps to use Docker Compose for Postgresql:

  1. Below is an example of the docker-compose.yml:

    services:
       database:
          image: 'postgres:latest'
          ports:
             - 5433:5432
          volumes:
             - D:\DockerVolumes\postgresql:/var/lib/postgresql/data/
          environment:
             POSTGRES_PASSWORD: lsf

  2. Move password to .env (optional)
    1. Remove or comment out the environment section from docker-compose.yml
    2. Add the following section:

      env_file:
           - .env

    3. Add a .env file beside the docker-compose.yml, with the following content:

      POSTGRES_PASSWORD=lsf

  3. Start the container by running:

    docker-compose up
